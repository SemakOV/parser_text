# parser_text


Описание: 

Модель пласта, который состоит из множества мелких ячеек, описывается специальным файлом данных.

Файл данных состоит из множества секций, каждая секция может быть вынесена в отдельный файл и

подключена с помощью ключевого слова INCLUDE. Так обычно поступают с секцией SCHEDULE, которая

содержит моделируемые операции и даты, на которые мы хотим получать отчеты. Часто приходится

заниматься автоматическим разбором содержимого этого файла, например, чтобы найти данные

вскрытия скважины, которые описываются ключевыми словами COMPDAT и COMPDATL.

Ниже приведен пример:



COMPDAT

'W1' 10 10  1   3       OPEN 1*    1     2     1     3*                1.0 /

/

 

COMPDATL

'W2' 'LGR3431' 10 10  2   2 OPEN 1*    1     2     1     3*                1.0918 /

/

 

Описание COMPDAT и COMPDATL можно найти в руководстве пользователя Eclipse 

 

Замечание: имя скважины, флаг и имя сетки - строки, координаты целые числа, все остальные параметры

вещественные числа. Какие-то параметры могут быть пропущены (*).

 

Задача:

-          Требуется разобрать приложенный файл test_schedule.inc и найти все секции COMPDAT/COMPDATL. Каждая

секция состоит из одной и более записей. Для одной скважины может быть множество записей. Каждая запись

состоит из 14 (COMPDAT) или 15 (COMPDATL) параметров, к которым мы хотим иметь доступ. Следует обратить

внимание, что мы хотим понимать на какую дату была записана секция и иметь возможность получить список

записей для некоторой скважины: "Какие есть вскрытия скважины W1 на 01.01.2019?".

-          Желательно наличие тестов.
